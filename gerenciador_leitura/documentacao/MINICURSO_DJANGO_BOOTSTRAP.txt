2.1 O QUE S√ÉO MODELS?
---------------------
Models s√£o CLASSES Python que representam TABELAS no banco de dados.
Cada atributo da classe = uma COLUNA na tabela.

EXEMPLO DO PROJETO:
```python

    nome = models.CharField(max_length=200)      # Coluna "nome" (texto)
    site = models.URLField(blank=True)           # Coluna "site" (URL)
```

Isso cria uma tabela chamada "editora" com 3 colunas!

2.2 TIPOS DE CAMPOS (FIELD TYPES)
----------------------------------
CharField(max_length=X)        ‚Üí Texto curto (at√© X caracteres)
TextField()                    ‚Üí Texto longo (sem limite)
EmailField()                   ‚Üí Email (valida formato)
URLField()                     ‚Üí URL (valida formato)
IntegerField()                 ‚Üí N√∫mero inteiro
PositiveIntegerField()         ‚Üí N√∫mero inteiro positivo
DecimalField(max_digits, decimal_places) ‚Üí Decimal (ex: dinheiro)
BooleanField()                 ‚Üí True/False
DateField()                    ‚Üí Data
DateTimeField()                ‚Üí Data + Hora
ImageField(upload_to="pasta/") ‚Üí Imagem (upload)
FileField(upload_to="pasta/")  ‚Üí Arquivo qualquer

PAR√ÇMETROS IMPORTANTES:
- blank=True ‚Üí Pode ficar vazio no formul√°rio
- null=True ‚Üí Pode ser NULL no banco
- default=valor ‚Üí Valor padr√£o
- unique=True ‚Üí Valor √∫nico (n√£o pode repetir)
- choices=lista ‚Üí Lista de op√ß√µes

2.3 RELACIONAMENTOS (FOREIGN KEY)
----------------------------------
ForeignKey: Relacionamento "muitos para um" (N:1)

EXEMPLO:

Isso significa: UM escritor pode ter V√ÅRIOS livros.
                 Mas CADA livro tem APENAS UM escritor.

on_delete=models.PROTECT ‚Üí N√ÉO permite deletar escritor se tem livros

3.1 O QUE S√ÉO VIEWS?
---------------------
Views s√£o FUN√á√ïES Python que:
1. Recebem uma requisi√ß√£o HTTP (request)

3.2 TIPOS DE VIEWS
------------------

A) FUNCTION-BASED VIEWS (FBV):

B) CLASS-BASED VIEWS (CBV):
```python

    template_name = 'livros.html'  # Define qual template renderizar

# - Automaticamente busca todos os Livros
# - Automaticamente renderiza o template
# - Menos c√≥digo, mas menos controle
```

PROJETO USA: Function-Based Views (mais simples e diretas)
Function-Based Views" (FBVs) s√£o fun√ß√µes Python simples que recebem uma requisi√ß√£o web
e retornam uma resposta HTTP. Elas s√£o um dos dois m√©todos principais
(sendo o outro as Class-Based Views, ou CBVs) para escrever a l√≥gica de visualiza√ß√£o
 em frameworks de desenvolvimento web, especialmente no Django. 

3.3 RENDER vs REDIRECT
-----------------------
render() ‚Üí Mostra um template HTML

```python

        # - √ötil ap√≥s salvar dados (evita duplica√ß√£o se F5)

        # - Processa template HTML
        # - Substitui {{ vari√°veis }} por valores
        # - Retorna HTML pronto pro navegador
```

3.5 QUERYSETS (BUSCAR NO BANCO)
--------------------------------
QUERYSETS s√£o como "perguntas" ao banco de dados.
Cada m√©todo retorna dados filtrados/ordenados.

Livro.objects.all()                    ‚Üí Todos os livros
    # O QUE FAZ: SELECT * FROM livro
    # RETORNA: Lista com TODOS os livros do banco
    
Livro.objects.filter(ano=2024)         ‚Üí Filtra por condi√ß√£o
    # O QUE FAZ: SELECT * FROM livro WHERE ano = 2024
    # RETORNA: Lista com livros publicados em 2024
    
Livro.objects.get(id=5)                ‚Üí Pega 1 objeto (erro se n√£o achar)
    # O QUE FAZ: SELECT * FROM livro WHERE id = 5 LIMIT 1
    # RETORNA: UM objeto Livro (n√£o √© lista!)
    # ATEN√á√ÉO: D√° erro se n√£o existir ou se existir mais de 1
    
Livro.objects.exclude(ano=2024)        ‚Üí Exclui por condi√ß√£o
    # O QUE FAZ: SELECT * FROM livro WHERE ano != 2024
    # RETORNA: Todos os livros EXCETO os de 2024
    
Livro.objects.order_by('-nota_media')  ‚Üí Ordena (- = decrescente)
    # O QUE FAZ: SELECT * FROM livro ORDER BY nota_media DESC
    # RETORNA: Livros ordenados da maior nota pra menor
    # SEM "-" seria ordem crescente (menor pra maior)
    
Livro.objects.count()                  ‚Üí Conta quantos
    # O QUE FAZ: SELECT COUNT(*) FROM livro
    # RETORNA: N√∫mero (int) com quantidade de livros

LOOKUPS (FILTROS AVAN√áADOS):
Lookups s√£o "modificadores" que v√£o depois de __ (dois underlines)

campo__exact=valor       ‚Üí Exatamente igual
    # titulo__exact='Python' ‚Üí t√≠tulo √© EXATAMENTE "Python"
    
campo__iexact=valor      ‚Üí Igual (ignora mai√∫sculas)
    # titulo__iexact='python' ‚Üí encontra "Python", "PYTHON", "python"
    
campo__contains=valor    ‚Üí Cont√©m (case-sensitive)
    # titulo__contains='Python' ‚Üí encontra "Aprenda Python"
    # N√ÉO encontra "aprenda python" (diferen√ßa mai√∫scula/min√∫scula)
    
campo__icontains=valor   ‚Üí Cont√©m (ignora mai√∫sculas)
    # titulo__icontains='python' ‚Üí encontra "Aprenda PYTHON", "python b√°sico"
    # MAIS USADO para buscas!
    
campo__in=[1,2,3]        ‚Üí Est√° na lista
    # ano__in=[2020, 2021, 2022] ‚Üí livros de 2020 OU 2021 OU 2022
    
campo__gt=10             ‚Üí Maior que (greater than)
    # paginas__gt=200 ‚Üí livros com MAIS de 200 p√°ginas
    
campo__gte=10            ‚Üí Maior ou igual
    # paginas__gte=200 ‚Üí livros com 200 ou mais p√°ginas
    
campo__lt=10             ‚Üí Menor que
    # preco__lt=50 ‚Üí livros com pre√ßo menor que R$50
    
campo__lte=10            ‚Üí Menor ou igual
    # preco__lte=50 ‚Üí livros at√© R$50 (incluindo R$50)
    
campo__startswith='A'    ‚Üí Come√ßa com
    # titulo__startswith='Python' ‚Üí "Python para Iniciantes", "Python Avan√ßado"
    
campo__endswith='Z'      ‚Üí Termina com
    # titulo__endswith='Brasil' ‚Üí "Hist√≥ria do Brasil", "Geografia do Brasil"
    
campo__isnull=True       ‚Üí √â NULL
    # editora__isnull=True ‚Üí livros SEM editora
    # editora__isnull=False ‚Üí livros COM editora

EXEMPLO:
```python

# O QUE FAZ:
# 1. Busca no banco todos os livros
# 2. Filtra os que t√™m "python" no t√≠tulo (ignora mai√∫scula/min√∫scula)
# 3. Retorna lista com resultados
# EXEMPLO: Encontra "Python B√°sico", "Aprenda PYTHON", "python avan√ßado"

# Busca livros com mais de 200 p√°ginas

# O QUE FAZ:
# 1. SELECT * FROM livro WHERE numero_paginas > 200

    titulo__icontains='python',  # Tem "python" no t√≠tulo
```

3.6 Q OBJECTS (QUERIES COMPLEXAS)
----------------------------------
Para fazer OR (ou) nas queries:

PROBLEMA: filter() normal s√≥ faz AND (e)
SOLU√á√ÉO: Use Q objects para OR (ou), NOT (n√£o)

```python
from django.db.models import Q

# Busca livros que tenham "Python" NO T√çTULO OU NO NOME DO ESCRITOR
livros = Livro.objects.filter(
    Q(titulo__icontains='python') |    # Condi√ß√£o 1: t√≠tulo tem "python"
    Q(escritor__nome__icontains='python')  # OU Condi√ß√£o 2: autor tem "python"
)
# O QUE FAZ:
# SELECT * FROM livro WHERE titulo LIKE '%python%' OR escritor.nome LIKE '%python%'
# RETORNA: Livros que tenham "python" no t√≠tulo OU livros de autor com "python" no nome
# EXEMPLO: Encontra "Python B√°sico" E "Livro de Jo√£o Python"

# EXEMPLO 2: Busca livros baratos OU bem avaliados
livros = Livro.objects.filter(
    Q(preco__lt=30) |           # Pre√ßo menor que R$30
    Q(nota_media__gte=4.5)      # OU nota maior/igual a 4.5
)
# Retorna: Livros baratos OU livros bons (ou ambos!)

# EXEMPLO 3: Combinar AND e OR
livros = Livro.objects.filter(
    Q(ano_publicacao__gte=2020) &  # Ano >= 2020 E
    (Q(titulo__icontains='python') | Q(titulo__icontains='django'))  # (Python OU Django)
)
# Retorna: Livros de 2020+ que tenham Python OU Django no t√≠tulo
```

OPERADORES:
| = OR (ou)    ‚Üí Pelo menos UMA condi√ß√£o precisa ser verdadeira
& = AND (e)   ‚Üí TODAS as condi√ß√µes precisam ser verdadeiras
~ = NOT (n√£o) ‚Üí Inverte a condi√ß√£o

EXEMPLO COM NOT:
```python
# Livros que N√ÉO s√£o de fic√ß√£o
livros = Livro.objects.filter(~Q(genero='Fic√ß√£o'))
# Equivalente a: WHERE genero != 'Fic√ß√£o'
```

```python
@staff_member_required
def editar_livro(request, livro_id):
    livro = get_object_or_404(Livro, pk=livro_id)
    if request.method == 'POST':
        form = LivroForm(request.POST, instance=livro)
        if form.is_valid():
            form.save()
            messages.success(request, 'Livro atualizado!')
            return redirect('lista_livros')
    else:
        form = LivroForm(instance=livro)
    return render(request, 'livro_form.html', {'form': form, 'livro': livro})
```
